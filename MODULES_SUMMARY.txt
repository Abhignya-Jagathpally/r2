================================================================================
DESIGN PATTERNS
================================================================================

1. CONSTRAINED AUTORESEARCH (Karpathy paradigm)
   - Preprocessing contract frozen with hash verification
   - Only architecture/hparams/training config tunable
   - Fixed budget: experiments, wall-clock time
   - Single metric: C-index
   - Full MLflow logging

2. PYTORCH LIGHTNING PATTERN
   - All models inherit from pl.LightningModule
   - Training/validation/test steps
   - Predict step for inference
   - Configure optimizers with scheduler

3. SURVIVAL ANALYSIS
   - Cox proportional hazards
   - Partial likelihood loss (handles censoring)
   - C-index metric (concordance)
   - Risk set computation

4. DOMAIN ADAPTATION
   - DANN with gradient reversal
   - CORAL loss for correlation alignment
   - Domain-invariant feature learning

5. MULTIMODAL FUSION
   - Late fusion (predict then combine)
   - Component model freezing
   - Three complementary strategies
   - Modality comparison

================================================================================
USAGE WORKFLOW
================================================================================

Phase 1: CLASSICAL BASELINE
  1. Load preprocessed data
  2. Create DeepSurv(pathway_dim=50)
  3. Train with Trainer
  4. Evaluate C-index on test set

Phase 2: FOUNDATION MODELS
  1. PathwayVAE for unsupervised representation
  2. DANN for cross-study alignment
  3. Extract aligned features

Phase 3: MULTIMODAL FUSION
  1. Train pathway + clinical models separately
  2. Combine with LateFusion (3 strategies)
  3. Or use MultimodalAttentionSurvival
  4. Compare C-index improvement

Phase 4: AUTORESEARCH OPTIMIZATION
  1. Define PreprocessingContract
  2. Create AutoresearchAgent
  3. Run agent.search() with Ray Tune
  4. Get best hyperparameters
  5. Train final model with agent.finalize()
